Geofence
a virtual perimeter surrounding a real-world area

Given this basic definition, we can assume a geofence can be imaged as a polygon that we define over a particular location. Furthermore, we can assume that for any given coordinates (lon, lat), those coordinates exist either within our outside our geofence. Because this operation (determining if a location is within a geofence) returns a true or false state (i.e., state of being), we can use this to trigger events (e.g., if someone’s location service places them in a particular location that happens to be within a geofence).

In this second scenario, the goal for this class is to create a web app that tells a story as the app user travels through a given area. With the app open, if the user enters one of your defined geofences, information will appear (e.g., to help guide them to the next geofence area). Obviously, we don’t want large polygons on a map that they are trying to find, but rather a story that unfolds as they traverse across a given landscape.

The Challenge

To create a geofence, I would like you to do some field data collection. Let’s try to utilize either Collector or QuickCapture for field collection of features.

Before we collect features in the field, we need a feature service, which I would like this as an opportunity for you to create your own field geodatabase and design fields and domains to assist your feature collection.

To create the geofencing app, one possibility is the Arcade/Javascript method, shown in this Geofencing app example.

The Communication

In this scenario, I would like each of you to communicate one thing you learn while attempting this project as either a vlog or a podcast. It should be informative and professional, similar to the “how to” instructions you produced in the first part of the class.

Additionally, I’d like each of you to share an ethics problem (or solution) that relates to geofencing one these platforms. It should reference a real-world scenario, requires citation, and a discussion based on your thoughts and opinions. Ideally, you should have a belief before researching a topic or scenario. From your research, your discussion should include whether or not your original belief was changed or enforced.

So, that’s two bits of information to share publicly and two platforms to choose from.
